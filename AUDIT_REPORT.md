# Shishka Clicker — code review and audit

## Что найдено

### Критичные/высокие
1. **Эксплойт покупки при быстром клике**
   - Цена и unlock-логика брались из уже рассчитанного UI-снимка `economy`, а не пересчитывались внутри `setState`.
   - При серии быстрых кликов один и тот же уровень можно было купить несколько раз по старой цене.
   - Исправлено: цена, текущий уровень и unlock теперь пересчитываются по актуальному `current` прямо внутри state-updater.

2. **Потеря точности экономики**
   - Игровой тик и ручной клик использовали `deriveEconomy()`, которая округляла значения до 0.1/0.01 для UI.
   - Из-за этого реальная экономика считалась с display-округлением и постепенно искажалась.
   - Исправлено: `deriveEconomy(state, { round: false })` теперь используется для runtime-логики; округление осталось только для UI.

3. **Небезопасная загрузка сохранения**
   - Save из `localStorage` раньше без валидации попадал в состояние.
   - Строки, `NaN`, отрицательные значения и поломанные объекты могли ломать баланс и расчёты.
   - Исправлено: добавлена нормализация чисел и карт уровней при merge сохранения.

### Средние
4. **Shared reference стартового состояния**
   - Возврат `STARTING_STATE` напрямую создавал риск разделяемой ссылки.
   - Исправлено: добавлен `createStartingState()` с глубоким клоном.

5. **Небезопасный доступ к `window.localStorage`**
   - Инициализация темы и пресетов падала бы в средах без `window`.
   - Исправлено: доступ обёрнут в безопасные helper-функции.

6. **Потенциальный stale update после unmount**
   - `setupDiscord().then(setUser)` мог сделать state update после размонтирования.
   - Исправлено: добавлен `isMounted` guard.

7. **Кнопки без `type="button"`**
   - Если приложение окажется внутри формы, кнопки могли бы триггерить submit.
   - Исправлено для основных action-кнопок.

8. **Пресеты темы сохранялись без защитного клонирования**
   - Исправлено: overrides при сохранении/применении теперь клонируются.

## Ограничения аудита
- Аудит был **статическим**: в контейнере не удалось установить npm/pnpm зависимости из-за отсутствия доступа к npm registry.
- Поэтому я не запускал полноценный `vite build` / `eslint`; исправления сделаны на основе ручного ревью и анализа кода.
