# Clicker Responsive Design

## Scope

Адаптировать экран кликера `clicker-screen--deck` под мобильные, планшетные и десктопные размеры без изменения игровой логики и без визуального отхода от текущего pixel UI.

## Goal

Сделать экран устойчивым на всех типовых ширинах:

- на телефоне экран читается без горизонтального скролла и без обрезания hero-блока
- на планшете сохраняется плотная, но чистая композиция
- на десктопе сохраняется текущий акцент на hero и панели прогресса

## Visual Thesis

Плотный аркадный control deck: крупный hero сверху или слева по ситуации, панель прогресса как единый пульт управления, без визуального шума и без сломанных переносов.

## Content Plan

1. Hero-блок с кнопкой клика остаётся главным визуальным якорем.
2. Deck-панель держит статус, табы, summary-статы и поле карточек.
3. На малых экранах порядок чтения идёт сверху вниз: hero, header/status, tabs, stats, field items.

## Interaction Thesis

1. Табы должны оставаться явными и доступными на любой ширине.
2. Hero должен сохранять крупную tappable область на мобильных.
3. Карточки прогресса должны перестраиваться без переполнения и без резких скачков высоты.

## Approach

Использовать CSS-first адаптацию с минимальными правками JSX:

- перестроить `clicker-deck-layout` под брейкпоинты mobile/tablet/desktop
- перевести `pixel-tabbar--deck` в более устойчивую grid/flex-композицию для узких экранов
- ужать `clicker-deck__header`, `clicker-deck__status`, `clicker-deck__stats` и типографику на малых ширинах
- при необходимости добавить точечные class/semantics в JSX, не меняя поведение вкладок
- стабилизировать сетку `ProgressFieldPanel` на узких экранах

## Constraints

- Не менять логику вкладок и блокировок секций.
- Не ломать существующие SSR-тесты разметки.
- Не трогать unrelated экраны.
- Не добавлять отдельную mobile-версию экрана.

## Testing

- Запустить targeted tests для `ClickerScreen` / `ProgressFieldPanel`.
- Запустить production build фронтенда, чтобы проверить CSS/JSX без синтаксических регрессий.

## Risks

- Слишком агрессивное уменьшение hero может ухудшить главный CTA.
- Изменение сетки табов может повлиять на длину русских лейблов.
- Поле карточек может требовать отдельного mobile-breakpoint сверх уже существующих стилей.
