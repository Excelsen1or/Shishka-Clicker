# Clicker Button Reset Design

## Scope

Восстановить `ClickerButton` до чистого и предсказуемого состояния, убрать искажение сцены и вернуть экрану `clicker-screen--deck` правильный порядок блоков: кнопка сверху, блок `Прогресс, покупки и мета-петля` снизу.

## Goal

- кнопка снова рендерится без сломанных пропорций фоновых слоёв и героя
- кнопка изолирована от layout-стилей deck-экрана
- CSS вокруг кнопки очищен от конфликтующих и дублирующихся правил
- deck-панель стабильно идёт ниже hero-кнопки на всех размерах

## Root Cause

- для scene-изображений используется растягивание через `object-fit: fill`, что ломает пропорции
- hero-контейнер и сама кнопка завязаны на несколько пересекающихся правил высоты и aspect-ratio
- адаптив deck-экрана и стили кнопки частично управляют одной и той же областью

## Approach

1. Вернуть `ClickerButton.jsx` к чистой сценовой структуре без лишних layout-зависимостей.
2. Оставить текущую игровую логику клика и эффектов, но не смешивать её с layout-логикой экрана.
3. Пересобрать связанный CSS-кластер кнопки:
   - сцена
   - hero-image
   - размеры контейнера
   - mobile/tablet overrides
4. Удалить или схлопнуть конфликтующие/дублирующие правила, которые мешают кнопке.
5. Вернуть `clicker-deck-layout` к вертикальному потоку: hero сверху, deck-панель снизу.

## Constraints

- не менять механику клика, эффектов и текстов кнопки
- не ломать существующие SSR-тесты
- не трогать unrelated экраны и не откатывать весь недавний адаптив

## Testing

- targeted test для `ClickerScreen` / `ProgressFieldPanel`
- production build фронтенда
- ручная проверка DOM-классов через SSR snapshot-подобные проверки

## Risks

- при грубом откате можно потерять полезные улучшения анимации
- при частичной зачистке CSS можно оставить скрытый конфликт в другом breakpoint
- кнопка и scene-слои требуют восстановления как одного блока, не по частям
